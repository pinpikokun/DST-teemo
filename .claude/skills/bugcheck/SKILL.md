# バグ・クラッシュチェック

コード変更のレビューを行い、バグやクラッシュの原因となりうる問題を検出する。

## 検証手順

### 第1ラウンド: 並行検証

複数のサブエージェントを起動し、以下の観点で並行にコードを検証する:

1. **ロジック安全性** — nil参照、未初期化変数、型不一致、境界条件
2. **レースコンディション** — タスクのタイミング依存、イベント順序依存
3. **エッジケース** — 攻撃可能な壁、海上、ボート上、洞窟（地上↔洞窟の切り替え）での挙動
4. **マルチプレイ** — サーバー/クライアント間の同期、他プレイヤーへの影響、ホスト/ゲスト間の差異

各エージェントは実際のコードを読み、具体的なファイル名・行番号を示して報告すること。

### 第2ラウンド: クロスチェック

第1ラウンドの結果を集約した後、別のエージェントに以下を検証させる:

- 検出された問題が**本当にバグか、偽陽性か**を、実際のコードを再読して判定する
- 偽陽性の場合はその理由を明示する
- 見落とされた問題がないかも確認する

単に並列で独立チェックするだけでは不十分 — 必ず相互検証を行うこと。

## スキップする既知事項（バグ報告不要）

以下は既知であり、意図的に対応しないと判断済み。報告しないこと:

- **`_noxiousTraps` リストのセーブ/ロード非永続化**: トラップは5分で自然消滅し、スタック回復も遅いため実害なし
- **Noxious Trap の `deployer` 参照のセーブ/ロード非永続化**: フォールバック検索で軽減済み
- **Flash の `IsPassableAtPoint` がボート上で動作しない**: 誤り。`IsPassableAtPoint` はデフォルトでボート（walkableplatform）を歩行可能として `true` を返す。DST本体の Lazy Explorer も同じ方法で判定しておりボート上で正常動作する
- **Flash の `busy` タグが即座に削除されて無意味**: AddTag/RemoveTag が同一フレーム内で同期実行されるため機能しないが、連打防止はクールダウンで保護済み。実害なし
